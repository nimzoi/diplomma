# Assignment 10 — Writing the Methodology and Technical Chapters

## 1. Objective of the Assignment

The purpose of this assignment is to transform the completed experimental work into a formally structured methodological and technical chapter of the thesis.

At this stage, the student must demonstrate:

* Ability to describe technical work in academic language
* Structured presentation of methodology
* Logical coherence between problem, design, and implementation
* Distinction between explanation and documentation

This assignment marks the beginning of the written integration of the practical component into the thesis document. The goal is not to describe "what was done" chronologically, but to explain how and why the methodology was constructed.

## 2. Scope of Work to Be Delivered

The student must submit:

* A draft of the Methodology chapter
* A draft of the Technical Implementation chapter
* Structured subsections covering data, models, evaluation, and environment

The total length should typically represent 50–70% of the final practical section of the thesis.

The text must include the following components:

### A. Methodological Framework

This section must formally present:

* The research problem (brief reference to Assignment 1)
* The experimental design (from Assignment 3)
* The evaluation strategy
* The comparison logic between models

The student must clearly articulate:

* What is being tested
* Under what conditions
* With what metrics

The structure should reflect scientific reasoning, not implementation order.

### B. Dataset Description

This subsection must present:

* Dataset characteristics
* Preprocessing summary
* Key constraints and limitations

The description must:

* Be analytical, not enumerative
* Avoid raw code details
* Clearly state data splits and validation strategy

If multiple datasets are used, justify their inclusion.

### C. Preprocessing and Feature Engineering

Describe:

* Data cleaning strategy
* Scaling and encoding procedures
* Feature engineering rationale
* Feature selection methods (if applied)

The focus must be on:

* Justification
* Reproducibility
* Impact on modeling

Avoid listing functions or libraries without explanation.

### D. Model Architecture and Configuration

Provide structured descriptions of:

* Baseline models
* Primary model
* Key hyperparameters
* Optimization strategy

The student must:

* Explain architectural decisions
* Justify model complexity
* Clarify evaluation fairness

Code fragments should only be included if they clarify methodology. The thesis is not a code repository.

### E. Experimental Environment

Describe:

* Software environment (libraries, versions)
* Hardware resources
* Reproducibility considerations

This section ensures technical transparency. It must confirm that results can be replicated under defined conditions.

### F. Writing Standards and Academic Tone

The student must ensure:

* Use of formal academic language
* Logical subsection structure
* Clear transitions between components
* Consistent terminology

Avoid:

* Informal narrative style
* Chronological storytelling
* Excessive reference to personal actions ("I implemented…") unless institutional standards allow first person

The writing must reflect structured engineering reasoning.

## 3. Academic Expectations for an Engineering Thesis

At the engineering level, the written methodology must demonstrate:

* Clarity of reasoning
* Structured technical explanation
* Justification of decisions
* Methodological transparency

The thesis is expected to:

* Show coherence between research question and experiments
* Provide reproducible technical detail
* Maintain balance between conciseness and completeness

It must avoid:

* Raw code dumps
* Superficial description of experiments
* Redundant explanation of standard algorithms without relevance

The emphasis must remain on design logic and experimental validity.

## 4. Quality Evaluation Criteria

Assignment 10 will be evaluated based on:

1. Logical structure and clarity
2. Coherence between chapters and previous assignments
3. Adequacy of methodological explanation
4. Academic writing quality
5. Consistency and precision of terminology

Approval of this assignment confirms that the thesis has entered its formal written phase.

## 5. Position Within the Seminar Continuum

* Assignments 1–9 established the full conceptual and experimental foundation.
* Assignment 10 formalizes this foundation into written academic structure.
* Assignment 11 will involve integrated review of the practical component, structural coherence assessment, and identification of missing elements.
* Assignment 12 will finalize conclusions, limitations, and future work directions.

At this stage:

* Major experimental work should be complete.
* Writing must reflect finalized results.
* Any structural inconsistency must be corrected before submission phase begins.

The thesis now transitions from experimentation to consolidation and refinement.
