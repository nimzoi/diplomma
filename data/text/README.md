# Extracted text corpus

This directory is intended for plain-text documents extracted from `data/raw/`.
It is generated by:

```bash
python3 scripts/extract_text_corpus.py --only-ok
```

For a smoke test:

```bash
python3 scripts/extract_text_corpus.py --only-ok --limit 10
```

## Purpose

The files in `data/raw/` contain the original downloaded artefacts: PDFs,
HTML pages and DOCX templates. They are suitable as source evidence, but not
ideal for retrieval-augmented generation directly. This directory stores the
text-only version of those documents so the next pipeline stage can perform:

1. parsing validation,
2. chunking,
3. embedding generation,
4. vector indexing,
5. retrieval evaluation.

## Output layout

```text
data/text/
  text_manifest.csv
  <source>/
    <document-stem>.txt
```

`text_manifest.csv` links each generated text file back to the original raw
file and preserves source, layer, format, topic IDs, title and content-audit
status.

## Notes

- PDF extraction requires the `pdftotext` command-line tool.
- HTML and DOCX extraction use only Python standard library modules.
- The script does not perform chunking, embeddings, vector indexing or semantic
deduplication.
- Use `--only-ok` to skip documents flagged as low quality by
  `data/content_audit.csv`.
