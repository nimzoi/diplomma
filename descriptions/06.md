# Task 06: Describing and Preparing ML Models

**Grading:** up to 10 points.

---

## 0. General structure for the “Model” chapter

For **every data type and every model**, try to keep a consistent structure, e.g.:

1. **Problem formulation**
2. **Data representation (after preprocessing)**
3. **Model selection rationale**
4. **Model architecture / structure**
5. **Hyperparameters**
6. **Training setup**
7. **Evaluation setup**
8. **Model interpretation & diagnostics**

Below I’ll show what each of these should contain in general, and then how to adapt them to **tabular / text / image / audio / video** specifically.

---

## 1. Common elements – what to always describe

### 1.1 Problem formulation

Explain clearly:

* **Input**: what is given to the model?
* **Output / target**: what should the model predict?
* **Task type**:

  * Classification (binary, multi-class, multi-label)
  * Regression
  * Sequence labeling
  * Ranking
  * Detection / segmentation (for images/video)
* **Loss function** you use:

  * Cross-entropy, MSE, MAE, etc.
* **Performance metrics**:

  * Accuracy, precision, recall, F1, ROC-AUC, IoU, BLEU, etc.
* **Why these metrics** are appropriate for your problem.

Example sentence:

> The goal of the model is to classify input X into one of K classes Y. Due to class imbalance and the importance of correctly identifying the minority class, F1-score and ROC-AUC were selected as primary evaluation metrics, with accuracy reported as a secondary metric.

---

### 1.2 Data representation (post-preprocessing)

Explain **how the preprocessed data looks when it enters the model**:

* Shape and type of input tensors
* Normalization or scaling (if relevant)
* Any special structures (e.g., sequences, images, spectrograms, clips, etc.)

Example:

> After preprocessing, each sample is represented as a vector of 42 numerical features (R⁴²). All features are standardized to zero mean and unit variance.

---

### 1.3 Model selection rationale

Explain **why you chose this model** (and not others):

* Theoretical reasons:

  * Linearity vs nonlinearity needs
  * Need for interpretability
  * Ability to handle high-dimensional data, etc.
* Practical reasons:

  * Training time and hardware limitations
  * Availability of pre-trained models
  * Simpler models as baselines

Example:

> Gradient Boosted Trees were selected due to their strong performance on structured tabular data, robustness to heterogeneous feature scales, and built-in handling of non-linear feature interactions.

This section can be shorter but should always exist.

---

### 1.4 Model architecture / structure

For **classical ML** (e.g., Random Forest, SVM):

* Describe the model conceptually:

  * “Ensemble of decision trees trained on bootstrap samples…”
  * “Maximum margin classifier with RBF kernel…”

For **neural networks / deep learning**:

* Explain:

  * Number and type of layers
  * Layer dimensions / feature map sizes
  * Activation functions
  * Special components (dropout, batch norm, attention, residual connections, etc.)
* Optionally include:

  * **Diagram / figure** of the network
  * **Table** summarizing layers (layer type, output shape, number of parameters)

Example:

> The network consists of three fully connected (dense) layers with 256, 128, and 64 units, respectively, each followed by a ReLU activation. To reduce overfitting, dropout with a rate of 0.5 is applied after the first and second dense layer.

---

### 1.5 Hyperparameters

List and justify key hyperparameters:

* For tree-based models:

  * Number of trees, max depth, learning rate, etc.
* For neural networks:

  * Learning rate, batch size, number of epochs
  * Optimizer (Adam, SGD, etc.)
  * Regularization (L2, dropout rate)
* For text models:

  * Vocabulary size, sequence length, embedding dimension
* For image/audio/video models:

  * Input resolution, frame rate, window length, etc.

Explain briefly:

* How hyperparameters were chosen:

  * Manual tuning
  * Grid search
  * Random search
  * Bayesian optimization
* What search space / ranges you used.

Example:

> Hyperparameters were tuned using grid search over the following ranges: number of trees ∈ {100, 200, 500}, maximum depth ∈ {4, 6, 8}, learning rate ∈ {0.01, 0.05}. The best configuration was selected based on F1-score on the validation set.

---

### 1.6 Training setup

Describe:

* Optimizer and learning rate schedule
* Batch size, number of epochs/iterations
* Train/validation/test split strategy

  * Random split, stratified split, time-based split, cross-validation
* Hardware:

  * CPU / GPU (optional but nice in an engineering thesis)
* Early stopping or other regularization during training

Example:

> The network was trained for up to 100 epochs using the Adam optimizer with an initial learning rate of 1e-3. Early stopping with patience of 10 epochs on the validation loss was applied to prevent overfitting. The batch size was set to 64.

---

### 1.7 Evaluation setup

Explain:

* How you evaluate:

  * Single train/val/test split vs k-fold cross-validation
* What exactly is reported:

  * Mean and standard deviation across folds
  * Confusion matrices
  * ROC curves, PR curves, etc.
* Any **baselines** you compare against.

Example:

> Model performance was assessed using 5-fold cross-validation on the training data and a final evaluation on a held-out test set. All reported metrics are averaged over the five folds, with standard deviations shown in parentheses.

---

### 1.8 Model interpretation & diagnostics

Depending on model type:

* Feature importance (for tabular)
* Saliency maps, Grad-CAM (for images)
* Attention weights (for text)
* Error analysis:

  * Typical failure cases
  * Confusion matrix analysis
* Overfitting check:

  * Training vs validation curves

---

## 2. Tabular data (structured data)

Tabular = data in rows and columns (numerical, categorical, binary).

### 2.1 Data representation (for tabular)

Describe:

* Number of features and their types:

  * Numerical: already scaled? log-transformed?
  * Categorical: one-hot / target encoding / embedding?
  * Binary flags
* Final input dimension:

  * Shape of feature vector

Example description:

> The final dataset contains 10,000 samples with 35 input features. 20 features are numerical and standardized to zero mean and unit variance. 15 categorical features are encoded using one-hot encoding, resulting in a total of 92 binary features.

---

### 2.2 Typical models for tabular data

You might describe one or several:

* **Linear models**:

  * Linear / Logistic Regression (possibly with L1/L2 regularization)
* **Tree-based models**:

  * Decision Tree
  * Random Forest
  * Gradient Boosting (XGBoost, LightGBM, CatBoost)
* **Simple feed-forward neural networks**:

  * When relationships are non-linear and data is large

For each model, explain:

* Why it fits tabular data
* Pros and cons (interpretability vs performance vs overfitting)

---

### 2.3 Specific hyperparameters to mention (tabular)

For **Random Forest** / **Gradient Boosting**:

* Number of trees (n_estimators)
* Max tree depth
* Minimum samples per split / leaf
* Learning rate (for boosting)
* Subsample / colsample (for boosting)
* Regularization parameters

For **Linear models**:

* Regularization type & strength (L1, L2, Elastic Net)
* Class weights (for imbalanced classification)

For **MLP (Neural Network)**:

* Number of layers and units
* Activations (ReLU, etc.)
* Dropout rate
* Batch size & learning rate

---

### 2.4 Evaluation & interpretation (tabular)

Include:

* Confusion matrix
* ROC / PR curves (if classification)
* Feature importance:

  * For tree models: built-in importance scores
  * For linear models: coefficients
  * Model-agnostic: SHAP, permutation importance
* Partial dependence plots (optional)

In your thesis, explain **how to read** these plots (at least briefly).

## 3. Text data (NLP)

Text can be: sentences, documents, reviews, etc.

### 3.1 Data representation (post-preprocessing)

Explain:

* Tokenization:

  * Word-level, subword (BPE, WordPiece), or character-level
* Vocabulary:

  * Size of the vocabulary
  * Handling of unknown words
* Sequence representation:

  * Fixed maximum sequence length (padding, truncation)
  * Embedding dimension

Examples of representations:

* Bag-of-words / TF-IDF vectors
* Word embeddings (Word2Vec, GloVe, fastText)
* Contextual embeddings (BERT, etc.)

Formal description:

> Each document is tokenized using a subword tokenizer (WordPiece) with a vocabulary of 30,000 tokens. Sequences are truncated or padded to a fixed length of 128 tokens. Tokens are mapped to 768-dimensional embeddings using a pre-trained BERT model.

---

### 3.2 Typical models for text

You can describe several types:

1. **Classical models**:

   * Logistic Regression / SVM on TF-IDF
   * Naive Bayes

2. **Sequence models**:

   * RNN (LSTM, GRU)
   * 1D CNN for text

3. **Transformer-based models**:

   * BERT / RoBERTa / DistilBERT, etc.
   * Fine-tuning vs feature extraction

For each, explain:

* How it processes sequences (order matters!)
* Advantages (context modeling, long range dependencies, etc.)

---

### 3.3 Architecture details (text)

For an **RNN**:

* Number of layers & hidden size
* Bidirectional or not
* Dropout
* How you aggregate sequence:

  * Last hidden state
  * Mean/max pooling
  * Attention

For a **Transformer** (e.g., BERT):

* Number of layers (if you describe architecture)
* Hidden size, number of heads (briefly)
* Whether you:

  * Fine-tune the whole model
  * Freeze it and only train a classifier head

Example description:

> The classifier is built on top of a pre-trained BERT-base model. The [CLS] token representation from the final transformer layer is passed through a dropout layer (p = 0.1) and a fully connected layer of size (768 × K), where K is the number of classes.

---

### 3.4 Hyperparameters & training (text)

Mention:

* Max sequence length
* Batch size (typically smaller for transformer models)
* Learning rate schedule (warmup, decay)
* Number of epochs (often small for fine-tuning, e.g. 3–5)
* Optimizer (AdamW commonly used for Transformers)

---

### 3.5 Evaluation & interpretation (text)

* Metrics:

  * Accuracy, F1 (macro/micro), especially with class imbalance
* Confusion matrix per class
* For multi-label tasks: micro/macro F1, subset accuracy, etc.
* Interpretation:

  * Feature importance over words (e.g., using LIME/SHAP)
  * Attention visualization (if using attention models)
  * Error analysis: examples of misclassified texts, ambiguous cases

Include a few examples in the thesis (e.g., correct vs incorrect predictions with commentary).

## 4. Image data

### 4.1 Data representation (post-preprocessing)

Explain:

* Image size and channels:

  * e.g., 224 × 224 × 3 (RGB)
* Normalization:

  * Per-channel mean/std normalization
* Data augmentation applied **during training** (describe briefly here or in preprocessing chapter, but reference it):

  * Random flips, rotations, crops, color jitter, etc.

Example:

> All images are resized to 224×224 pixels and normalized using the mean and standard deviation statistics of the ImageNet dataset. During training, random horizontal flips and random crops are applied as data augmentation.

---

### 4.2 Typical models for images

You can describe:

* **Classical ML** (for small images / features):

  * Extracted features (SIFT, HOG) + SVM
* **Convolutional Neural Networks**:

  * Simple CNN defined by you
  * Pre-trained networks (ResNet, VGG, MobileNet, EfficientNet, etc.) with transfer learning

---

### 4.3 Architecture details (images)

If you design your own CNN, specify:

* Number of convolutional layers
* Filter sizes and number of channels
* Pooling (max/avg)
* Fully connected layers at the end
* Activation functions

If you use a **pre-trained model**:

* Which architecture (e.g., ResNet-50)
* How you adapt it:

  * Replace final classification layer for your number of classes
  * Fine-tune all layers or only last layers

Example:

> The base architecture is ResNet-18 pre-trained on ImageNet. The final fully connected layer is replaced by a new linear layer mapping from 512 features to 3 output classes. All layers are fine-tuned using a reduced learning rate for the pre-trained layers and a higher rate for the new classification head.

---

### 4.4 Hyperparameters & training (images)

Mention:

* Input size (e.g., 224×224)
* Batch size (depends on GPU memory)
* Learning rate and schedule
* Optimizer (SGD with momentum, Adam)
* Data augmentation settings (intensity, probability)
* Number of epochs

---

### 4.5 Evaluation & interpretation (images)

* Metrics:

  * Accuracy, F1, top-k accuracy, IoU (for segmentation)
* Visualizations:

  * Confusion matrix
  * Example predictions (correct/incorrect)
* Model interpretation:

  * Saliency maps / Grad-CAM to highlight which regions of the image influence the prediction
* Error analysis:

  * Show typical misclassified examples and describe patterns (blurred images, occlusions, etc.)

## 5. Audio data

### 5.1 Data representation (post-preprocessing)

Audio must be described carefully:

* Sampling rate (e.g., 16 kHz, 44.1 kHz)
* Channels (mono / stereo)
* Segment length (e.g., 1-second windows)
* Representation:

  * Raw waveform
  * **Spectrogram** (magnitude of STFT)
  * Mel-spectrogram
  * MFCC features

Example:

> The audio signals are sampled at 16 kHz and converted to mono. Each sample is segmented into 1-second windows with 50% overlap. For each window, a 64-bin Mel-spectrogram is computed, resulting in a 64×128 time-frequency representation.

This is important because it defines the input shape for your model.

---

### 5.2 Typical models for audio

Depending on representation:

* On **MFCC / hand-crafted features**:

  * Classical models (SVM, Random Forest)
  * Simple feed-forward networks
* On **spectrograms (2D)**:

  * CNNs similar to image models (treat spectrogram as an image)
* On **raw waveform / sequences**:

  * 1D CNNs
  * RNNs
  * Transformer for audio

Explain:

* Why you chose spectrogram vs raw waveform (trade-off between simplicity and information content)

---

### 5.3 Architecture details (audio)

If using spectrograms + CNN:

* Describe kernel sizes:

  * Along time, frequency, or both
* Pooling strategy:

  * In time, frequency, or 2D
* How you aggregate over time to get a fixed-size representation:

  * Global average pooling
  * Attention over time

If using sequences + RNN:

* Sequence length (number of frames)
* Hidden size, number of layers, bidirectionality

---

### 5.4 Hyperparameters & training (audio)

Mention:

* Window length and hop length in STFT (if used)
* Number of Mel filters (for Mel-spectrograms)
* Model hyperparameters (as for other modalities)
* Data augmentation:

  * Time masking, frequency masking (SpecAugment)
  * Adding noise, pitch shift, time stretch (describe if used)

---

### 5.5 Evaluation & interpretation (audio)

* Metrics similar to classification/regression tasks
* Confusion matrix
* Visualization:

  * Spectrograms of correctly / incorrectly classified signals
* Interpretation:

  * Which frequency ranges or time regions are important (e.g., by occluding parts of spectrogram or using Grad-CAM-like methods)

## 6. Video data

Video is essentially **sequence of images** (with time).

### 6.1 Data representation (post-preprocessing)

Explain:

* Frame rate:

  * Original and after downsampling (e.g., use 10 fps instead of 30 fps)
* Spatial resolution:

  * e.g., frames resized to 112×112 or 224×224
* Clip length:

  * How many consecutive frames per sample (e.g., 16, 32)
* Whether you:

  * Use **raw frames** as input (3D tensor: T × H × W × C)
  * Extract frame-level features (e.g., using CNN) and then aggregate over time
  * Use optical flow or other motion features

Example:

> Videos are downsampled to 10 frames per second and resized to 112×112 pixels. For each sample, a 16-frame clip is extracted, resulting in an input tensor of shape (16, 112, 112, 3).

---

### 6.2 Typical models for video

Several main strategies:

1. **Frame-wise CNN + temporal aggregation**:

   * Apply 2D CNN on each frame
   * Aggregate features using:

     * Average pooling over time
     * RNN over frame features
     * Temporal attention

2. **3D CNNs**:

   * Convolutions over space + time simultaneously
   * e.g., C3D, I3D, R(2+1)D

3. **Two-stream architectures**:

   * One stream for RGB frames
   * One stream for optical flow (motion)
   * Fuse predictions

Explain in your thesis which approach you use and why (performance vs complexity).

---

### 6.3 Architecture details (video)

If using 3D CNN:

* Number of 3D conv layers
* Kernel sizes (temporal dimension + spatial)
* Pooling over time & space
* Final classifier head

If using CNN + RNN:

* CNN for frame feature extraction (e.g., ResNet)
* Feature dimension for each frame
* RNN type (LSTM/GRU) and hidden size
* How you obtain final prediction (e.g., last hidden state, aggregated features)

---

### 6.4 Hyperparameters & training (video)

Mention:

* Clip length and sampling strategy (random start, uniform sampling)
* Batch size (often much smaller due to memory)
* Learning rate, number of epochs
* Data augmentation:

  * Spatial: flips, crops
  * Temporal: random clip start, dropping frames

---

### 6.5 Evaluation & interpretation (video)

* Metrics (usually classification)
* How you aggregate predictions:

  * Per clip → per video (average scores across multiple clips from the same video)
* Interpretation:

  * Visualizing key frames contributing to decision
  * Grad-CAM applied on selected frames
* Error analysis:

  * Typical failure cases (motion blur, fast motion, similar classes)

## 7. How to present everything nicely in the thesis

### 7.1 Recommended tables

For each modality/model, add tables such as:

1. **Model architecture summary table**
   Columns: Layer #, Type, Output Shape, #Params

2. **Hyperparameters table**
   Columns: Hyperparameter, Value, Notes

3. **Results table**
   Rows: Different models or settings
   Columns: Accuracy, Precision, Recall, F1, etc.

### 7.2 Recommended figures

* Model architecture diagrams (especially for deep models)
* Training/validation loss & metric curves
* Confusion matrices
* Examples of:

  * Input samples with model predictions
  * Saliency/attention maps (for image/audio/video/text)

### 7.3 Suggested paragraph templates

You can adapt these in your thesis:

**Model introduction:**

> In this section, we describe the machine learning models used to solve the classification task on [data type] data. After preprocessing, each input sample is represented as [description of representation]. We evaluate several model architectures, including [model list], and select the final model based on performance on a validation set.

**Architecture description:**

> The proposed model consists of [N] layers. First, [description of early layers]. Then, [description of middle layers]. Finally, the output of the last hidden layer is passed through a fully connected layer producing [K] logits corresponding to the [K] classes. A softmax function is applied to obtain class probabilities.

**Training setup:**

> The model is trained using the [optimizer name] optimizer with an initial learning rate of [value] and batch size [value]. The training runs for up to [N] epochs, with early stopping based on the validation [metric]. All experiments are conducted on a [hardware description].
